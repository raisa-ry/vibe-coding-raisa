## Project: HikePack AI

AI-powered hiking gear list generator — Spring Boot 3.4 + vanilla JS single-page app.

## Project Setup

- **IDE**: JetBrains IntelliJ IDEA
- **Language**: Java 21 (LTS)
- **Build**: Maven (`pom.xml` at repo root)
- **VCS**: Git (remote: `https://github.com/raisa-ry/vibe-coding-raisa.git`)

## Key Commands

```bash
# Terminal 1 — MCP server (stub mode, no API key needed)
cd mcp-server && npm install && node index.js

# Terminal 1 — MCP server with Claude AI parsing
cd mcp-server && ANTHROPIC_API_KEY=sk-ant-... node index.js

# Terminal 2 — Spring Boot (stub mode)
"/Applications/IntelliJ IDEA.app/Contents/plugins/maven/lib/maven3/bin/mvn" spring-boot:run

# Terminal 2 — Spring Boot with Claude AI generation
ANTHROPIC_API_KEY=sk-ant-... "/Applications/IntelliJ IDEA.app/Contents/plugins/maven/lib/maven3/bin/mvn" spring-boot:run

# Build fat JAR
"/Applications/IntelliJ IDEA.app/Contents/plugins/maven/lib/maven3/bin/mvn" package -DskipTests

# Run JAR
java -jar target/hikepack-ai-1.0.0.jar
```

**Note:** System `mvn` is not on PATH; use IntelliJ's bundled Maven at the path above, or open in IntelliJ and use the Maven panel.

## Architecture

```
mcp-server/
  package.json   # Node.js deps (@modelcontextprotocol/sdk, express)
  index.js       # MCP HTTP server on :3001 — parse_hike_conditions tool

src/main/java/com/hiking/
  HikingPackListApplication.java     # Spring Boot entry point
  controller/PackListController.java # POST /api/generate, /api/parse-description, /api/location
  service/PackListService.java       # Claude API call + prompt builder
  service/HikeParserService.java     # MCP JSON-RPC client (initialize → tools/call)
  service/LocationService.java       # Geocoding + weather fetch
  model/TripInput.java               # Request body for /api/generate
  model/ParsedConditions.java        # Response body for /api/parse-description
  model/LocationData.java            # Response body for /api/location
  model/PackListResponse.java        # Response (categories, what-ifs, tips)
  model/Category.java
  model/WhatIfScenario.java
  exception/LocationNotFoundException.java

src/main/resources/static/
  index.html   # Single-page form + results
  style.css    # Earth-tone responsive styles + print media
  app.js       # Form → API → render; parse description; share link (URL hash); print
```

### Request flow

```
Browser → POST /api/parse-description
  → HikeParserService (MCP client): initialize → tools/call
    → mcp-server/index.js → Claude API → ParsedConditions JSON
  → form fields auto-filled with highlight animation

Browser → POST /api/location → LocationService → LocationData
Browser → POST /api/generate → PackListService → Claude API → PackListResponse
```

### Temperature handling

The form displays **°C**. `collectInputs()` converts to °F before sending to the Java backend (which uses °F internally). The MCP server and location preview also return/display °C.

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | No | Anthropic API key; both Spring Boot and MCP server fall back to logic-based stubs if unset |
| `SERVER_PORT` | No | Spring Boot port override (default 8080) |

MCP server always runs on port 3001. If it is unreachable, `POST /api/parse-description` returns 500 with an inline error; the rest of the app is unaffected.

## Features

1. **AI description parsing** — type a plain-English hike description; MCP server extracts location, date, duration, temperature, weather, terrain, and experience level; form fields auto-fill with a highlight animation
2. **Location fetch** — geocode a place name or coordinates, pull real elevation and weather forecast
3. **Trip inputs** — temperature range (°C), weather checkboxes (rain/wind/snow/heat), terrain, duration, experience level
4. **AI pack list** — 5 gear categories generated by `claude-sonnet-4-6`
5. **What-if scenarios** — Cold Snap, Sudden Storm, Injury, Getting Lost (each with extra gear + actions)
6. **Pro tips** — trip-specific safety/comfort advice
7. **Shareable link** — form state base64-encoded in URL hash (`#share=...`); auto-generates on load
8. **Print view** — `@media print` hides form, expands all scenarios, formats for paper
